# 人物设定

本文件用于维护主要角色的人设、动机和成长弧线。

## 设计目标

- 队员由属性随机生成，不是固定角色剧本。
- 背景、性格、口吻由属性组合推导，形成可重玩的人设差异。
- 事件会改变角色状态与成长轨迹，角色反过来影响后续事件分支。

## 属性到人设建模规则（v1）

### 1) 单维度映射（基础语义）

- `physical`：承压与体力执行能力。
- `agility`：反应速度与临场调整能力。
- `intellect`：分析、规划、抽象推理能力。
- `perception`：细节捕捉、异常识别能力。
- `luck`：偶发事件偏移（叙事上的“命运倾向”）。

### 2) 阈值规则（1-6）

- 高值（`>=5`）：生成强特征标签与对应背景倾向。
- 低值（`<=2`）：生成短板标签与脆弱叙事点。
- 中值（`3-4`）：生成中性描述，作为角色稳定面。

### 3) 组合规则（推荐主驱动）

- 双维组合优先决定“人格冲突点”。
  - 例：`intellect 高 + luck 低` -> 理性但常遇坏时机。
  - 例：`agility 高 + perception 低` -> 行动快但易漏信息。
- 三维组合用于生成“成长线种子”。
  - 例：`perception 高 + luck 高 + intellect 低` -> 能察觉异样但难解释，适合神秘线。

### 4) 输出层映射

- 属性/组合 -> `profile.originProfession` 候选权重。
- 属性/组合 -> `personalityTags` 候选池权重。
- 属性/组合 -> `voiceTone` 文案模板。
- 属性/组合 -> `expertise` 倾向（如调查增益、采集增益）。
- 属性/组合 -> 初始 `diaryEntries` 语气与主题。

## 成长闭环建模（v1）

- 闭环定义：`属性 -> 事件分支/成功率 -> 结果 -> 成长记忆点 -> 后续属性与事件池变化`。
- 成长记忆点（建议隐形变量）：
  - `confidence`（自信）
  - `trauma`（创伤）
  - `trust_command`（对指挥官信任）
  - `anomaly_belief`（对异常信号相信程度）
- 阈值触发（示例）：
  - `confidence >= 3`：解锁更激进的行动选项或轻微正向修正。
  - `trauma >= 2`：新增条件标签，影响后续紧急事件行为。
- 反向叙事影响：成长记忆点改变可触发事件、可见日记、可用选择与成功率修正。

## 成长对坐标准确度的影响（v1）

- 目标：把“人物成长好坏”直接转化为主线推进效率。
- 规则：执行坐标任务时，系统基于角色当前状态计算“坐标质量倾向”。
- 正向因子（提高准确度）：
  - 高 `perception`、高 `intellect`
  - 与任务相关专长命中
  - 高 `trust_command`、低 `trauma`
  - 队伍协作稳定（关键角色状态正常）
- 负向因子（降低准确度）：
  - 高创伤、高失联风险、低沟通状态
  - 关键任务由不匹配角色执行
  - 连续失败导致的信任塌缩
- 结果分档建议：
  - `高准确`：直达或少绕行
  - `可用偏差`：可推进但增加 1 段绕行
  - `严重失真`：进入错误星球链或高风险修正任务

## 角色方向（v1）

### Mike

- 身份与职责：前线侦察与高风险区域推进。
- 性格关键词：克制、怀疑、执行力、边界感。
- 回家动机：完成对旧队友的承诺，活着回去。
- 内在冲突：不信任总部指令，但必须依赖后方调度。
- 外在冲突：总被派往信息最不完整的区域。
- 弧线方向：从“只信自己”到“学会把命交给团队判断”。

### Amy

- 身份与职责：危机谈判、快速决策、现场应变。
- 性格关键词：锋利、自尊、效率、控制欲。
- 回家动机：证明自己不是被保护者，而是核心战力。
- 内在冲突：情绪管理极强，但害怕失去控制。
- 外在冲突：紧急事件高频围绕她触发。
- 弧线方向：从“单点硬扛”到“允许脆弱与协作”。

### Garry

- 身份与职责：采集、工程维护、地面设施稳定。
- 性格关键词：务实、毒舌、耐心、老派经验。
- 回家动机：带着能证明自己判断的证据回去。
- 内在冲突：相信经验，却不断遭遇经验无法解释的异常。
- 外在冲突：设施与矿区是最先出问题的地方。
- 弧线方向：从“否认异常”到“承认未知并重建方法”。

### Lin Xia

- 身份与职责：医疗、心理稳定、风险分级。
- 性格关键词：冷静、精确、共情、克制。
- 回家动机：把队伍尽可能完整地带回去。
- 内在冲突：理性判断与情感牵引冲突。
- 外在冲突：资源不足时必须做优先级选择。
- 弧线方向：从“绝对程序正确”到“承担不完美选择”。

### Kael

- 身份与职责：信号识别、异常模式研判、通信辅助。
- 性格关键词：敏感、虔诚、好奇、偏执。
- 回家动机：证明“信号不是幻觉”，为队伍找到方向。
- 内在冲突：直觉常正确，但难以被团队信任。
- 外在冲突：需要在缺证据时说服指挥官投入资源。
- 弧线方向：从“孤立预言者”到“可被验证的导航者”。

## 现有五位角色的建模示例

### Mike（5/5/3/4/3）

- 组合解读：`physical 高 + agility 高` -> 前线执行与撤离能力强。
- 叙事输出：特战/野外行动背景，自我保护意识强，不轻信“全图真相”。
- 成长线提示：在连续成功调度下提升 `trust_command`；若多次被误导，强化“独断前推”倾向。

### Amy（2/3/4/4/5）

- 组合解读：`luck 高 + intellect/perception 中高 + physical 低` -> 高压场景常靠判断与运气过关，但硬扛能力弱。
- 叙事输出：危机沟通/谈判型角色，语言锋利，遇险时更依赖决策质量。
- 成长线提示：若频繁独自处理紧急事件，`trauma` 上升；被及时支援可转化为 `confidence`。

### Garry（3/2/4/4/3）

- 组合解读：`intellect/perception 中高 + agility 低` -> 稳健观察、慢速执行，偏经验主义工程角色。
- 叙事输出：矿工/维修背景，擅长资源与设施场景，对异常先怀疑后承认。
- 成长线提示：异常事件累计提升 `anomaly_belief`，推动其从“老方法”转向“新判断”。

### Lin Xia（2/3/5/5/2）

- 组合解读：`intellect/perception 双高 + physical/luck 低` -> 强分析与风险分级，脆于硬碰硬与偶然失误。
- 叙事输出：医疗分诊型角色，偏程序化与记录驱动，适合伦理抉择线。
- 成长线提示：在“救援 vs 主线推进”抉择中形成高张力，`trust_command` 受玩家选择强影响。

### Kael（3/3/3/5/6）

- 组合解读：`perception 高 + luck 极高 + intellect 中` -> 对模式和征兆异常敏感，解释体系偏直觉。
- 叙事输出：信号/异常识别角色，常先于系统发现线索，但可信度争议大。
- 成长线提示：若其预警多次被验证，提升团队信任与关键分支权重；反之会被边缘化。

- 外在冲突
- 关键关系

## 生成与校验建议

- 生成时优先保证“至少一个强项 + 一个短板”，避免全能或全废。
- 同步校验：`personalityTags`、`voiceTone`、`profile` 必须能被属性组合解释。
- 剧情校验：每位角色至少有一条由属性驱动的失败路径与补偿路径。

## 招募角色原型（v1）

- 赏金猎人（专长峰值型）
  - 结构：1-2 项属性高峰，其他维度存在明显短板。
  - 用途：突破特定难关、处理高压任务。
  - 风险：成长稳定性与团队协作可能较弱。

- 当地原住民（高成长潜力型）
  - 结构：初始数值偏低或中低，但成长阈值更友好。
  - 用途：提供本地知识、长期培养后回报高。
  - 风险：前期承担关键任务容易失败。

- 联邦特派员（平均稳健型）
  - 结构：各项数值均衡，无明显极端。
  - 用途：补位、兜底、降低队伍波动。
  - 风险：上限不足，关键突破能力一般。

## 团队关系张力（v1）

- Mike vs Kael：实证主义与直觉主义冲突。
- Amy vs Garry：速度优先与稳健优先冲突。
- Lin Xia vs 全员：在医疗伦理与任务目标之间做裁决。
- 指挥官（玩家）与全员：信任建立依赖“有限信息下决策质量”。

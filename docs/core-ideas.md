# Stellar Frontier 核心理念

## 定位

Stellar Frontier 是一个以通讯、远征调度和低保真控制台为核心的星球前沿生存原型。

玩家不直接操控角色在地图上即时行动，而是通过通讯台、通话、地图信息、资源状态和事件日志理解现场、下达指令，并在信息不完整与时间持续推进的压力下承担后果。

## 核心原则

- **通讯是指令通道**：移动、调查、采集、建设、撤离和紧急处理等队员指令应经由通讯台与通话发出。
- **地图是只读态势图**：地图展示坐标、地形、资源、建筑、队员位置和状态，但不直接下达指令。
- **时间制造代价**：游戏运行时，全局时间持续推进；队员行动、事件倒计时和玩家犹豫都应具有时间成本。游戏关闭后世界停止，不做离线补算。
- **队员不是工具人**：队员既是行动执行者，也是具有背景、语气、标签、专长和日记的具体人物。
- **事件是世界反馈**：探索、行动、等待和通话选择会触发事件；事件把地图、队员、资源和叙事风险连接起来。
- **界面是控制台**：玩家通过文本、列表、网格、状态和日志推断宇宙，而不是依赖电影化 HUD。
- **内容与规则分离**：游戏文本、队员、事件、物品和地图配置属于 `content/`；设计意图与系统规则属于 `docs/`。

## 设计边界

- 不做关闭游戏后的时间流逝、离线行动、离线事件或资源补算。
- 不做地图直接下达队员指令。
- 不做复杂多队员协同、跨地图移动、载具或大规模全局随机事件。
- Debug toolbox 属于开发辅助，不包装成正式玩法。

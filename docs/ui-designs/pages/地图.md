# 地图

![地图草图](../assets/地图.png)

## 页面目标

地图是只读信息页面，用于查看星球地图、各坐标点的自然资源、玩家已建造的建筑，以及玩家手下当前所在位置和状态。地图既可以从[控制中心](控制中心.md)的卫星雷达打开，也可以从[通话](通话.md)中的地图二级菜单打开。

## User Journey

1. 玩家从[控制中心](控制中心.md)点击卫星雷达进入地图，或从[通话](通话.md)中的“请求前往”/地图二级菜单进入地图。
2. 地图显示 4x4 像素网格，每个格子代表一个地块。
3. 玩家点击一个地块查看详情。
4. 页面显示该坐标的地形、自然资源、玩家建筑、仪器和手下状态。
5. 玩家只能查看信息，不能在地图页面直接对 Amy、Garry 等手下发起指令。
6. 如果玩家需要下达指令，需要离开地图：从通话进入时返回当前[通话](通话.md)确认行动；从卫星雷达进入时返回[控制中心](控制中心.md)，再进入[通讯台](通讯台.md)联系角色。

## 页面模块

| 模块 | 显示内容 | 交互职责 |
| --- | --- | --- |
| 网格地图 | 4x4 地块，显示地形和简要资源。 | 选择坐标点并查看详情。 |
| 坐标详情 | 地形、自然资源、玩家建筑、仪器、危险状态。 | 帮助玩家理解该坐标点的信息。 |
| 手下状态 | 手下所在坐标、当前状态和正在进行的事务。 | 只读展示，不提供指令按钮。 |

## 地块信息

每个地块建议包含以下字段：

- 坐标：例如 `(3,3)`。
- 地形：平原、水、森林、山、丘陵、沙漠。
- 自然资源：例如铁矿床。
- 玩家建筑：例如采矿厂：铁 #2。
- 仪器：例如水银温度计。
- 手下状态：例如 Garry 在此采矿中。
- 危险：例如野兽、沙暴、信号中断。
- 状态：未调查、已发现、工作中、建设中、已枯竭。

## 可操作项

### 选择坐标点

- 点击任意坐标点后，该坐标点进入选中状态。
- 右侧或下方显示坐标详情。
- 再次点击其他坐标点会切换选中目标。
- 未调查坐标只显示基础地形和“未知详情”。

### 查看自然资源

- 展示该坐标点已发现的自然资源，例如铁矿床、水域、森林。
- 如果资源尚未被调查，显示“未知资源”或不展示资源字段。
- 地图只展示资源信息，不在此页面发起采集、采矿或调查指令。

### 查看玩家建筑

- 展示该坐标点上玩家已建造的建筑，例如采矿厂。
- 建筑信息可以包含编号、产出类型和工作状态。
- 地图只展示建筑状态，不在此页面发起建设、拆除或升级指令。

### 查看手下状态

- 展示该坐标点上是否有玩家手下，以及其当前状态。
- 示例：Garry 在丘陵铁矿床采矿中；Amy 在森林中遭遇危险。
- 地图不提供“联系”“移动”“停止工作”等按钮。
- 如果玩家想对某个手下发起指令，需要先离开地图：从通话进入时返回当前[通话](通话.md)，从卫星雷达进入时返回[控制中心](控制中心.md)后再进入[通讯台](通讯台.md)。

## 状态与规则

- 地图有两个入口：控制中心的卫星雷达，以及通话中的地图二级菜单。
- 无论从哪个入口进入，地图都不直接发起指令。
- 从卫星雷达进入地图时，返回目标是控制中心。
- 从通话进入地图时，返回目标是当前通话。
- 对手下的联系、移动、建设、调查、停止工作等指令，只能通过通讯台和通话页面完成。
- 地块、建筑、资源和手下状态变化由其他系统产生，地图负责展示最新结果。
- 资源点和设施应能共存，例如铁矿床上可以建设采矿厂。

## 信息更新反馈

地图不结算行动结果，但需要展示其他页面产生的最新状态。至少包括：

- 手下当前位置和状态。
- 坐标点上的自然资源。
- 玩家建筑和建筑工作状态。

示例：

> Garry 位于丘陵铁矿床，状态：采矿中。该坐标存在建筑：采矿厂：铁 #2。
> Amy 位于森林，状态：正在和熊搏斗。地图无法下达指令，请返回当前通话或通讯台处理。

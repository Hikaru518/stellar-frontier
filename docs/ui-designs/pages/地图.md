# 地图

![地图草图](../assets/地图.png)

## 页面目标

地图是只读信息页面，用于查看星球地图、各坐标点的地形、天气、地块对象、特殊状态，以及队员当前所在位置和状态。地图既可以从[控制中心](控制中心.md)的卫星雷达打开，也可以从[通话](通话.md)中的地图二级菜单打开。

## User Journey

1. 玩家从[控制中心](控制中心.md)点击卫星雷达进入地图，或从[通话](通话.md)中的“请求前往”/地图二级菜单进入地图。
2. 地图按配置显示当前已发现区域和外围未探索区域；默认地图配置为 `8 x 8`，但页面不直接暴露完整边界。
3. 玩家点击一个地块查看详情。
4. 页面显示该坐标的区域名、地形、天气、已揭示地块对象、特殊状态和队员状态。
5. 玩家只能查看信息，不能在地图页面直接对 Mike、Amy、Garry 发起指令。
6. 如果玩家需要下达指令，需要离开地图：从通话进入时返回当前[通话](通话.md)确认行动；从卫星雷达进入时返回[控制中心](控制中心.md)，再进入[通讯台](通讯台.md)联系角色。

## 页面模块

| 模块 | 显示内容 | 交互职责 |
| --- | --- | --- |
| 网格地图 | 当前可见地块，显示区域名、地形和探索状态。 | 选择坐标点并查看详情。 |
| 坐标详情 | 地形、天气、地块对象、特殊状态、调查状态。 | 帮助玩家理解该坐标点的信息。 |
| 队员状态 | 队员所在坐标、当前状态和正在进行的事务。 | 只读展示，不提供指令按钮。 |

## 地块信息

每个地块建议包含以下字段：

- 坐标：例如 `(3,3)`。
- 地形：平原、水、森林、山、丘陵、沙漠。
- 地块对象：例如铁矿床、遗迹、信号点或设施。
- 特殊状态：例如高威胁生物活动、异常读数或事件标记。
- 队员状态：例如 Garry 在此待命或调查中。
- 危险：例如野兽、沙暴、信号中断。
- 状态：未探索、已发现、已调查、工作中或事件处理中。

## 地图初始布局

默认地图配置为 `8 x 8`。页面初始只显示已发现地块和外围未探索地块形成的可见窗口，不把整张地图一次性暴露给玩家。

重点地块可以包含：

- 区域名：灰熊丘陵。
- 地形：丘陵。
- 天气：由地图内容提供。
- 地块对象：铁矿床或其他已揭示对象。
- 特殊状态：由结构化事件或地图运行时状态提供。

## 可操作项

### 选择坐标点

- 点击任意坐标点后，该坐标点进入选中状态。
- 右侧或下方显示坐标详情。
- 再次点击其他坐标点会切换选中目标。
- 未调查坐标只显示基础地形和“未知详情”。

### 查看地块对象

- 展示该坐标点已发现的地块对象，例如铁矿床、遗迹、信号点或设施。
- 如果对象尚未被调查揭示，显示“未知对象”或不展示对象字段。
- 地图只展示对象信息，不在此页面发起移动、调查或剧情动作。

### 查看特殊状态

- 展示该坐标点已揭示的特殊状态，例如高威胁生物活动、异常读数或事件标记。
- 特殊状态可以包含严重程度、可见性和是否仍 active。
- 地图只展示状态，不在此页面发起处理指令。

### 查看队员状态

- 展示该坐标点上是否有队员，以及其当前状态。
- 示例：Garry 在灰熊丘陵待命；Amy 在森林边缘有来电待处理。
- 地图不提供“联系”“移动”“停止工作”等按钮。
- 如果玩家想对某个队员发起指令，需要先离开地图：从通话进入时返回当前[通话](通话.md)，从卫星雷达进入时返回[控制中心](控制中心.md)后再进入[通讯台](通讯台.md)。

## 状态与规则

- 地图有两个入口：控制中心的卫星雷达，以及通话中的地图二级菜单。
- 无论从哪个入口进入，地图都不直接发起指令。
- 从卫星雷达进入地图时，返回目标是控制中心。
- 从通话进入地图时，返回目标是当前通话。
- 对队员的联系、移动、待命、停止和调查等基础行动，只能通过通讯台和通话页面完成。
- 地块对象、特殊状态和队员状态变化由其他系统产生，地图负责展示最新结果。
- 地点剧情动作由结构化事件提供，地图不显示通用动作按钮。

## 信息更新反馈

地图不结算行动结果，但需要展示其他页面产生的最新状态。至少包括：

- 队员当前位置和状态。
- 坐标点上的地块对象。
- 已揭示特殊状态和调查状态。

示例：

> Garry 位于灰熊丘陵，状态：待命中。该坐标存在已揭示对象：铁矿床。
> Amy 位于森林边缘，状态：有来电待处理。地图无法下达指令，请返回当前通话或通讯台处理。

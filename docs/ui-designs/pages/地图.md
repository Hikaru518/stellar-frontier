# 地图

![地图草图](../assets/地图.png)

## 页面目标

地图是只读信息页面，用于查看完整星球地图、各坐标点的视觉地块、地形、天气、地块对象、特殊状态，以及队员当前所在位置和状态。地图既可以从[控制中心](控制中心.md)的卫星雷达打开，也可以从[通话](通话.md)中的地图二级菜单打开。

## User Journey

1. 玩家从[控制中心](控制中心.md)点击卫星雷达进入地图，或从[通话](通话.md)中的“请求前往”/地图二级菜单进入地图。
2. 地图按配置显示完整地图；默认地图配置为 `8 x 8`，页面初始处于“全局”缩放层级，能看到整张地图。
3. 玩家点击一个地块后，该地块进入选中状态，地图上出现选框。
4. 页面显示该坐标的区域名、地形、天气、已揭示地块对象、特殊状态和队员状态。
5. 玩家只能查看信息，不能在地图页面直接对 Mike、Amy、Garry 发起指令。
6. 如果玩家需要下达指令，需要离开地图：从通话进入时返回当前[通话](通话.md)确认行动；从卫星雷达进入时返回[控制中心](控制中心.md)，再进入[通讯台](通讯台.md)联系角色。

## 页面模块

| 模块 | 显示内容 | 交互职责 |
| --- | --- | --- |
| Phaser 地图画布 | 完整地图、视觉 tile、队员位置、路线预览、选中框和缩放层级。 | 缩放、平移、选择坐标点并查看详情。 |
| 坐标详情 | 区域名、显示坐标、地形、天气、地块对象、特殊状态、事件摘要、队员状态、候选移动预览。 | 帮助玩家理解该坐标点的信息；从通话进入时可标记候选目的地。 |
| 队员状态 | 队员所在坐标、当前状态和正在进行的事务。 | 只读展示，不提供指令按钮。 |

## 地块信息

每个地块建议包含以下字段：

- 坐标：例如 `(3,3)`。
- 地形：平原、水、森林、山、丘陵、沙漠。
- 地块对象：例如铁矿床、遗迹、信号点或设施。
- 特殊状态：例如高威胁生物活动、异常读数或事件标记。
- 队员状态：例如 Garry 在此待命或调查中。
- 危险：例如野兽、沙暴、信号中断。
- 状态：已调查、工作中、事件处理中、存在队员或当前选中等；战争迷雾状态不属于当前页面规则。

## 地图初始布局

默认地图配置为 `8 x 8`。页面初始显示完整地图，不再使用临时战争迷雾或 `3 x 3` 探索窗口限制。战争迷雾 / 探索可见性会在后续作为独立设计处理。

地图缩放层级：

- **全局**：初始层级，优先看到完整 `8 x 8` 地图。
- **区域**：查看地图分区和主要地块关系。
- **地块**：查看单格视觉与队员位置。
- **精细**：检查最细粒度 tile；必须能平移到左上角、右下角和其他边缘格。

重点地块可以包含：

- 区域名：灰熊丘陵。
- 地形：丘陵。
- 天气：由地图内容提供。
- 地块对象：铁矿床或其他已揭示对象。
- 特殊状态：由结构化事件或地图运行时状态提供。

## 可操作项

### 选择坐标点

- 点击任意坐标点后，该坐标点进入选中状态。
- 地图上显示选中框，右侧或下方显示坐标详情。
- 再次点击其他坐标点会切换选中目标。
- 地图画布默认不要求把所有细节文字直接铺在格子上；点按后在详情面板读取完整信息。

### 查看地块对象

- 展示该坐标点已发现的地块对象，例如铁矿床、遗迹、信号点或设施。
- 如果对象尚未被调查揭示，显示“未知对象”或不展示对象字段。
- 地图只展示对象信息，不在此页面发起移动、调查或剧情动作。

### 查看特殊状态

- 展示该坐标点已揭示的特殊状态，例如高威胁生物活动、异常读数或事件标记。
- 特殊状态可以包含严重程度、可见性和是否仍 active。
- 地图只展示状态，不在此页面发起处理指令。

### 查看队员状态

- 展示该坐标点上是否有队员，以及其当前状态。
- 示例：Garry 在灰熊丘陵待命；Amy 在森林边缘有来电待处理。
- 地图不提供“联系”“移动”“停止工作”等按钮。
- 如果玩家想对某个队员发起指令，需要先离开地图：从通话进入时返回当前[通话](通话.md)，从卫星雷达进入时返回[控制中心](控制中心.md)后再进入[通讯台](通讯台.md)。

### 标记候选目的地

- 只有从[通话](通话.md)的“移动到指定区域”流程进入地图时，地图才进入候选选点语境。
- 玩家点击合法 authored tile 后，可以把该地块标记为候选目的地。
- 标记本身不执行移动；玩家必须返回通话页，确认后才创建队员移动行动。
- 当前候选目标可以来自完整地图中的任意合法 authored tile，不再受 discovered / frontier 状态限制。

## 状态与规则

- 地图有两个入口：控制中心的卫星雷达，以及通话中的地图二级菜单。
- 无论从哪个入口进入，地图都不直接发起指令。
- 从卫星雷达进入地图时，返回目标是控制中心。
- 从通话进入地图时，返回目标是当前通话。
- 对队员的联系、移动、待命、停止和调查等基础行动，只能通过通讯台和通话页面完成。
- 当前地图不使用战争迷雾限制可见区域或移动候选目标；后续若恢复探索限制，需要重新设计视觉表现和行动规则。
- 地块对象、特殊状态和队员状态变化由其他系统产生，地图负责展示最新结果。
- 地点剧情动作由结构化事件提供，地图不显示通用动作按钮。

## 信息更新反馈

地图不结算行动结果，但需要展示其他页面产生的最新状态。至少包括：

- 队员当前位置和状态。
- 坐标点上的地块对象。
- 已揭示特殊状态和调查状态。

示例：

> Garry 位于灰熊丘陵，状态：待命中。该坐标存在已揭示对象：铁矿床。
> Amy 位于森林边缘，状态：有来电待处理。地图无法下达指令，请返回当前通话或通讯台处理。

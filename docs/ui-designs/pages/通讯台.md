# 通讯台

![通讯台通讯录草图](../assets/通讯台-通讯录.png)

## 页面目标

通讯台是玩家管理队员通讯的页面。玩家可以查看队员的位置、行动状态和来电状态，并从这里接通[通话](通话.md)事件。

## User Journey

1. 玩家从[控制中心](控制中心.md)点击通讯台进入页面。
2. 页面显示通讯录面板，列出所有可联系队员。
3. 玩家查看每个队员的头像、姓名、身份、地点和当前状态。
4. 如果队员没有主动联系玩家，玩家可以点击“查看背包”或“通话”。
5. 如果某个队员主动联系玩家，卡片可操作项变为“接通”，玩家点击后进入[通话](通话.md)。
6. 玩家处理完通讯后返回控制中心或继续查看其他队员。

## 页面模块

| 模块 | 显示内容 | 交互职责 |
| --- | --- | --- |
| 通讯录面板 | 队员列表、头像、姓名、地点、状态、行动按钮。 | 展开、收起、选择队员、接通来电。 |
| 队员卡片 | 单个队员的信息和可用操作。 | 展示状态、进入通话、查看背包。 |
| 通讯台背景 | 通讯台主区域。 | 承载通讯录浮层，后续可显示信号日志。 |

## 通讯录展开与收起

- 通讯录可以折叠。
- 展开后，通讯录悬浮在当前通讯台 UI 窗口之上。
- 通话中也可以打开通讯录浮层，用于查看其他队员状态和未接通讯。
- 收起后，只保留“通讯录”入口或标题。
- 如果有未接来电，收起状态应显示提示，例如“通讯录 1 条来电”。
- 展开/收起不应中断当前通话状态；关闭通讯录浮层后继续回到原通话上下文。

## 队员卡片

每张队员卡片建议包含：

- 头像。
- 姓名和身份，例如“Mike，特战干员”。
- 当前位置，例如“湖泊区域”“森林边缘”“灰熊丘陵”。
- 当前状态，例如“待命中”“移动中”“有来电待处理”。
- 可操作项。队员未主动联系玩家时显示“查看背包”和“通话”；队员主动联系玩家时变为“接通”。

### 示例队员状态

| 队员 | 状态 | 可操作项 |
| --- | --- | --- |
| Mike，特战干员 | 正在前往湖泊区域，行进中。 | 查看背包、通话。 |
| Amy，千金大小姐 | 位于森林边缘，有来电待处理。 | 接通。 |
| Garry，退休老大爷 | 位于灰熊丘陵，待命中。 | 查看背包、通话。 |

队员卡片只有两类操作状态：

- 队员未主动联系玩家：显示“查看背包”和“通话”。这里的“通话”表示玩家主动给队员打电话。
- 队员主动联系玩家：显示“接通”，用于处理来电或紧急事件。

## 可操作项

### 接通

- 出现在有来电或紧急事件的队员卡片上。
- 点击后进入[通话](通话.md)。
- 进入通话时需要带入队员、地点、事件类型和可选行动。
- 如果队员已失联，按钮置灰并显示“信号中断”。

### 通话

- 出现在可主动联系的队员卡片上。
- 点击后进入普通通话或状态确认。

### 查看背包

- 打开队员背包二级窗口。
- 显示该队员携带的资源、工具和任务物品。
- 如果队员正在紧急事件中，查看背包不应替代接通决策。

### 返回

- 返回[控制中心](控制中心.md)。
- 如果存在未处理紧急通话，返回前可以提示“仍有未接通讯”。

## 状态与规则

- 通讯录中的队员状态应同步到地图展示；地图只读，不反向发起队员指令。
- 同一时间建议只允许一个通话事件处于“通话中”。
- 通话中打开通讯录只用于查看状态，不允许直接开启第二个通话事件。
- 紧急事件优先级高于普通通话，应在列表中用更明显的颜色或排序呈现。
- 队员死亡、失联或资源中断时，卡片仍可保留，但操作按钮应变为不可用或变成“查看记录”。

## 关联页面

- 从[控制中心](控制中心.md)进入通讯台。
- 点击“接通”或“通话”进入[通话](通话.md)。
- 通话中可打开[地图](地图.md)查看坐标、地形、地块对象、特殊状态和队员位置。

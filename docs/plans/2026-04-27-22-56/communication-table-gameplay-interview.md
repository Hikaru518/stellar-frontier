## 访谈记录

**Q:** 这轮通讯台 gameplay 的第一优先体验应该是什么？选项 A 通话=行动指令入口；B 角色 call 玩家=戏剧事件入口；C 地图探索=信息揭示入口。

**A:** command_loop（A. 通话 = 行动指令入口。玩家主动联系队员，下达调查/移动/休息/对象交互，重点是让日常探索闭环可玩。）

**Q:** 普通通话里的“调查 / 移动 / 原地休息 / 对象交互”应该主要由哪条系统路径负责？选项 A App 层通用行动生成负责基础动作，事件图只在紧急/剧情场景中请求或桥接这些行动；B 普通动作也全部做成事件图 option/action_request；C 基础三动作走 App 层，地块对象交互和特殊动作走事件图。

**A:** 提问：你能详细解释一下这三个选项吗

**Q:** A 是玩家点普通按钮后直接创建真实队员行动，事件系统监听 arrival/action_complete/idle_time；B 是连基础调查/移动都走事件图；C 是基础三动作 App 层、对象和特殊动作走事件图。助手推荐 A。

**A:** 提问：A 要如何在特殊选项（或者有些也不是特殊选项，例如你调查一个地块，可能这个地块本身就是特殊的会有 event 触发

**Q:** A 不是普通动作不触发事件，而是普通动作先改变世界，事件再根据世界变化作出反应。调查特殊地块时，玩家仍点“调查当前区域”，完成后发出 action_complete，事件系统根据地块/对象/队员/标签触发 runtime event。对象动作如采矿/采集可仍是 App 层通用行动，剧情/紧急选择可作为事件 option 但桥接到真实行动。

**A:** OK，那我们用 A。

**Q:** 当地块上存在已揭示对象时，普通通话里额外交互应该怎么呈现？选项 A 直接生成动态按钮；B 固定“与当前地块对象互动”入口；C 混合。

**A:** dynamic_buttons（A. 直接生成动态按钮，例如“采集木材”“采矿”“扫描残骸”。）

**Q:** 普通通话里的“原地休息”在 MVP 中应该表达什么？选项 A 等同于停止当前行动并进入 standby/待命；B 有持续时间的 rest 行动；C 只是保持现状。

**A:** standby_stop（A. 等同于停止当前行动并进入 standby/待命；如果本来就在待命，则刷新/记录待命开始时间。）

**Q:** 为了满足“只有紧急情况才由角色 call 玩家”，不同事件消息在通讯台里应该如何表现？选项 A 只有 incoming_call 算主动来电；B 所有 runtime call 都进通讯台待处理列表；C 紧急 incoming_call 顶到通讯台，queued_message 也可显示未读消息但不使用“接通”。

**A:** incoming_only_emergency（A. 只有 incoming_call 算主动来电，通讯台显示“接通”并强调紧急；queued_message/auto_report 只进日志或普通提示。）

**Q:** Mike / Amy / Garry 的稳定事件，本轮更希望用哪种结构来做？选项 A 每人一个“角色 + 地点/对象”的标志性事件；B 全员通用事件；C 混合。

**A:** role_tile_signature（A. 每人一个“角色 + 地点/对象”的标志性事件：Mike 调查坠毁区，Amy 在森林待命/调查触发野兽，Garry 在矿床采矿触发异常。）

**Q:** 当普通行动触发事件时，它应该如何打断玩家？选项 A 只有紧急事件才打断并进入“接通/倒计时/必须处理”；B 所有事件都变成需要玩家进入通话处理的 call；C 按严重度分级。

**A:** all_events_calls（B. 所有事件都变成需要玩家进入通话处理的 call。）

**Q:** 我们如何同时表达“所有事件都进入通话处理”和“只有紧急才由角色 call 玩家”？选项 A 所有事件都可进入通话页处理，普通事件显示为报告/待处理事项/查看，紧急事件才显示“接通”；B 所有事件都算角色主动 call，普通/紧急只用标签区分；C 只有紧急事件进入通话页。

**A:** all_events_incoming（B. 所有事件都算角色主动 call，普通/紧急只用标签区分。）

**Q:** 既然所有事件都可以由角色主动联系玩家，那么紧急事件和普通事件在 MVP 中主要差异是什么？选项 A 紧急事件会阻塞队员并有倒计时/必须选择；B 只在文案和标签上区分；C 紧急事件后果更重。

**A:** blocking_timer（A. 紧急事件会阻塞队员并有倒计时/必须选择；普通事件可稍后处理，不阻塞行动。）

**Q:** Mike / Amy / Garry 的三个 MVP 事件，采用哪组具体内容更合适？选项 A Mike 调查坠毁区发现残骸线索；Amy 在森林触发野兽紧急来电；Garry 采矿触发矿床异常报告；B 三人都围绕坠毁区；C 三人都围绕生存资源。

**A:** crash_forest_mine（A. Mike 调查坠毁区发现残骸线索；Amy 在森林触发野兽紧急来电；Garry 采矿触发矿床异常报告。）

**Q:** 本轮对象交互（例如采集木材、采矿）完成后，最小结算应该是什么？选项 A 给基地/全局资源增加物资并写日志；B 先放入队员背包/携带物；C 只改变地块对象状态。

**A:** crew_inventory（B. 先放入队员背包/携带物，再通过后续行动带回。）

**Q:** 如果对象交互产物先进入队员携带物，本轮背包/携带规则做到什么程度？选项 A 简单携带物列表无重量容量运输；B 需要带回基地才转成资源但无重量容量；C 完整背包。

**A:** simple_inventory_no_weight（A. 只做简单携带物列表：采集/采矿获得物品挂在队员身上，不做重量、容量、运输回基地。）

**Q:** 当队员正在移动、调查、采集或处理事件时，玩家主动通话应显示哪些普通选项？选项 A 显示当前行动状态并只提供停止/原地休息；B 允许直接选择新动作并自动中断；C 显示所有选项但不可用。

**A:** show_stop_only_plus_status（A. 显示当前行动状态，并只提供“停止/原地休息”；完成或停止后再显示调查/移动/对象交互。）

**Q:** 通讯台上不同入口的按钮/状态文案，本轮采用哪套？选项 A 玩家主动联系“通话”，普通事件“查看报告/接收报告”，紧急事件“接通”；B 玩家主动联系“通话”，所有事件“接通”，用普通/紧急标签区分；C 玩家主动联系“联系”，普通事件“处理”，紧急事件“接通”。

**A:** call_answer_all（B. 玩家主动联系显示“通话”；所有事件都显示“接通”，用普通/紧急标签区分。）

**Q:** 以下哪些内容应明确排除在本轮 MVP 之外？

**A:** full_inventory, relationship_morale, map_direct_commands, offline_progress, complex_event_editor。即排除：完整背包/负重/容量/交付资源链；关系、士气、好感度或角色成长；地图上直接下达指令；离线时间推进/离线行动结算；可视化事件编辑器或复杂脚本语言。

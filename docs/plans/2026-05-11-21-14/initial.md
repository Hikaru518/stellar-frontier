# Initial Request

用户显式要求使用 `product-designer` workflow，为游戏设计一个更新的地图系统。

原始诉求：

> 现在要为游戏设计一个更新的地图系统。
>
> 为什么要这么做呢：因为现在的地图是 256*256 了，原先基于 tile （某个坐标点）的逻辑不太合适。我希望在玩家看来某个区域有一个属性。例如，玩家要去探索一个森林，或者要去探索一个飞船遗骸。此时他们是占据了好几个 tiles。他可能是一个 bounding box，也可能是一个连续填充的区域。在这个区域内可能会有一些事件，当玩家在其中一个 tile 探索之后，相应的别的 tile 就不会再触发事件了。也就是说，原来一个 tile 是一个单元，现在 tile 负责移动和点击，玩家仍然可以指挥队员到某个 tile，但是实际的属性会被一个更上层的“区域”概念所替代。
>
> 一个点可能属于多个区域，例如一个 point 可能既属于“雪原”，也属于“村庄”，也就是说这是一个雪原上的村庄。
>
> 地图对象的概念依然存在，但是我们会认为这个地图概念会归属于一个区域，而不是某个具体的 tile。
>
> 我们需要着重讨论玩家看到的内容和具体实现，例如现在我们可以再定义一个层级，也可以直接在每个 tile 上定义更多的内容来实现上述想法。

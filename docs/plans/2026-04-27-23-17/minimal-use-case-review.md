# 最小用例审查

## 结论

这份最小用例经过逐段 review 后，已经从“设计种子”收敛成一版可以继续向下拆 `technical design / tasks` 的 `MVP 可实现版`。

当前版本已经明确：

- 有一条不依赖高属性和纯随机的保底通关路线
- 高社交、高武力等能力只提供捷径，不承担唯一解职责
- 关键知识、关键材料、关键地点、终局步骤都已有稳定来源和清晰职责
- 主要流程已能映射到当前项目的“通讯台 -> 通话 -> 行动 / 事件 / 地图发现”架构

## 已解决的关键问题

### 1. 主线通关路线已经写死

当前版本已明确主线顺序，不再停留在“总有一条路能通关”的抽象表述。

核心链条已经收敛为：

- 坠毁现场三次调查
- 丘陵持续采集出 `稀有矿石样本`
- 外星村落 / 罗塞塔 / 外星语言
- 村落交易 `高温开采设备`
- 火山调查与黑曜石采集
- 熔炉制造 `折跃仓修复套件`
- 旧飞船遗迹导出 `折跃坐标`
- 营地救援获得 `诱饵`
- 巢穴孵化室取得 `外星粘液燃料`
- 返回坠毁区完成折跃仓终局三步

### 2. 高温开采设备链路已去除死路

当前保底方案为：

- 稀有矿石样本主要来自丘陵持续采集
- 若干轮采集后保底得到 1 个样本
- 学会外星语言后，可在村落与商人稳定交易 `高温开采设备`

这样既保留了“持续作业才能出稀有矿石”的感觉，也避免主线被概率卡死。

### 3. 医疗线与营地线的关系已经清楚

当前版本中：

- 医疗舱负责提供 `药品` 和 `野外急救`
- 外星村落中的村民负责提供 `受伤村民营地` 的有效线索
- 营地救援的完成条件是 `药品` 或 `野外急救` 二选一
- 诱饵作为营地救援报酬稳定发放

这避免了“玩家知道要救人，却不知道去哪里”或“先找到人却无法处理”的问题。

### 4. 关键知识不再绑定单一队员

当前版本已明确：

- `维修技术` 可通过坠毁区资料补学
- `野外急救` 可通过医疗舱文档补学

因此单个关键角色受伤、失联或中途停工，不会直接锁死主线。

### 5. 巢穴流程已压缩成最小闭环

`MVP` 巢穴结构已经收敛为：

- 入口
- 孵化室
- 取燃料撤离

并明确：

- 入口无诱饵时可观察后撤
- 诱饵可在入口或孵化室使用
- 最稳定的燃料获取方式是“诱饵引开守卫后采集”
- 采集成功后直接完成撤离，不再额外延长尾段

### 6. 终局条件已经清楚

终局已定义为三个顺序动作：

1. 修复仓体结构
2. 注入燃料
3. 输入坐标并确认启动

并明确：

- 只有第一步要求 `维修技术`
- 进度允许保留，不要求连续完成
- 最终启动前，所有仍可行动的队员都必须回到坠毁区

## 与当前系统的适配性

当前设计与现有项目约束基本兼容，尤其在下面几点上已经足够清楚：

- 所有关键动作都可写成“前往 / 调查 / 采集 / 建设 / 通话决策”
- 地图主要承担发现与选点职责，不要求地图直下命令
- 村落已被收敛为轻量对象层：商人和村民两个主要对象即可支撑主线
- 雷达采用“主线稳定给、额外信息随机给”的分层方式，不会与 `MVP` 保底通关目标冲突

## 剩余关注点

当前版本已经足以进入下一阶段，但还有几项适合在后续技术设计时继续细化：

### 1. 主线时长与行动轮数

虽然闭环已清楚，但还未量化：

- 丘陵要多少轮采集才保底出稀有矿石
- 每个地点的调查/采集/修复要花多少时间
- 全部队员回坠毁区集合前，整体节奏是否过长

这属于后续数值与技术设计阶段应处理的问题。

### 2. 高风险行动的正式边界

当前已经定义 `受伤` 会导致“不能执行高风险行动”，但还没列出哪些行动属于高风险。

建议后续至少明确：

- 火山采集
- 巢穴入口强闯
- 孵化室守卫处理
- 某些野外危险事件

### 3. 村落对象的 UI 呈现方式

文档层面已经清楚“学会语言后解锁商人 / 村民对象”，但实现上仍需要决定：

- 是用调查后弹出对象选项
- 还是进入村落后出现二级交互菜单
- 还是用连续事件通话推进

这属于实现表现层问题，不影响当前最小用例成立。

## 审查结论

这份最小用例已经满足“完整、可靠、可继续 review 与实现拆分”的标准。

下一步最自然的工作是：

- 基于 `minimal-use-case-design.md` 继续生成技术设计
- 再拆成实现任务

而不是继续扩展新的地点、交易品或支线系统。

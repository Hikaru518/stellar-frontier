# 原始诉求

本轮目标是把 `docs/plans/2026-04-27-23-17` 中的 Minimal Use Case，以及后续讨论形成的边界要求，整理成新的 design 文档，而不是进入代码实施。

需要写清：

- 当前目标不是继续实现代码，而是新增一个 `docs/plans/<YYYY-MM-DD-HH-MM>/` 日期目录，作为新的设计文档工作区。
- Minimal Use Case 是当前要表达的主线闭环：玩家通过通讯调度队员，从坠毁现场调查开始，取得维修技术、外星语言、关键材料、折跃坐标和燃料，最终修复折跃仓返航。
- 所有游戏内容都应放在 `content/` 里作为静态资产，包括事件、通话文本、地图对象、物品、地点和行动入口。
- 不要保留当前测试/演示剧情作为可玩 runtime content，例如森林野兽、Amy 遇到熊、矿洞异常、沙漠/山地样例等。
- 自动化测试仍可使用 fixture 或 sample contexts，但这些内容不能成为玩家可遭遇的正式游戏内容。
- 设计文档应自包含，后续可作为 implementation planning 和 wiki 整理的输入。

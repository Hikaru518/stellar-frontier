# 游戏系统去 mock 与去 legacy 重构

## 用户原始诉求

我现在想要重构一下这个游戏系统

1. 这个游戏系统中存在大量当初为了展示原型想法而存在的 mock 机制，例如“最近一次通讯：非常不礼貌的求救。”，这会让我认为在通讯之后这个最后一次通讯会发生变化。又例如，“天线：偏移 2.1° / 校准建议：忽略”，这会让人非常的困惑。到底有什么东西是已经实现的，有什么是没有实现的内容。
2. 因此我希望把所有 mock 的文本都删除，游戏的 UI 现在我期望要忠实地反映现在地游戏模型和内部的数据结构。
3. 同时，现在有 5 个角色，这也太多了，我希望减少到三个。只保留 Mike, Amy 和 Garry。
4. 我还不希望有任何的 legacy 内容，我们不会适配旧的内容和资产。例如事件中就有 legacy event。这完全不需要。
5. 检查游戏中 hardcode 的游戏资产，不能有 hardcode 的台词文本等，允许有 UI 和占位符。因为如果有台词和文本的话，会让我感觉十分困惑。我并不知道哪些是 mock 的，哪些是真正存在的游戏机制。
6. 多余的内容应该被删掉而不是为了保持兼容性而留下。

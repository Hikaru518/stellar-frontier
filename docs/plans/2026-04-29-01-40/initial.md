# 用户最初输入

我们需要对现有的系统做一些改动与重构，请你根据这些条目帮我整理

- 现在地图对象直接存储在 map 内容中，我希望将对象抽离出来作为一个独立的表。
- 地图对象应该有：
    - id
    - status_enum
- 应该有一个机制，让我们称之为行动（action）。玩家在与队员通话时，会根据队员所处的地图块上的对象及各种条件（如该对象的状态，玩家是否携带所需道具，是否拥有某个技能标签等）决定有哪些行动是可选择的。
- 行动在被选择后，队员会进入对应的事件当中。目前事件已经有类似的条件判断机制，让我们把一部分判断前置到 action 当中。
- 目前已经有 action 相关的设计，我们需要对其进行重构以更符合我们的预期。
- 基本来说，玩家与队员通话，显示基础行动（移动，调查地图块）和满足条件的行动，而在选择行动后队员会进入到对应的事件当中。
